# 🧬 Obscura — Drug Toxicity Prediction AI

> **CodeCure AI Hackathon · Track A · Drug Toxicity Prediction (Pharmacology + AI)**
>
> **Team Bit by Bit (B3)** — Subhagata Sardar · Hrishita Ray · Aishwariya Saha

Obscura is a production-grade machine learning pipeline for predicting drug toxicity across all **12 Tox21 assay targets**. It uses an ensemble of LightGBM, XGBoost, and Random Forest models trained on ~7,500 comprehensive molecular features, with SHAP-based explainability and a dynamic Streamlit prediction interface.

---

## 📋 What Does Obscura Do?

Obscura predicts whether a chemical compound is likely to be toxic based on its **molecular structure (SMILES string)**. Given a molecule, it:

1. Standardizes and parses the SMILES into a molecular graph
2. Engineers ~7,564 features (fingerprints + physicochemical descriptors + ADMET flags)
3. Runs the features through an AUPR-weighted ensemble of three gradient boosting models
4. Outputs a **toxicity probability** and a **risk level** (LOW → HIGH)
5. Breaks down predictions across all **12 Tox21 assay targets** individually
6. Explains the prediction using **SHAP feature importance**

### What "Toxicity" Means Here

Obscura predicts **Tox21 assay-based toxicity** — specifically whether a compound activates:

- **Nuclear receptors (NR):** NR-AR, NR-ER, NR-AhR, NR-Aromatase, NR-ER-LBD, NR-PPAR-gamma
- **Stress response pathways (SR):** SR-ARE, SR-MMP, SR-p53, SR-HSE, SR-ATAD5

This covers **acute receptor-mediated and genotoxic toxicity**. It does not cover chronic mechanisms like carcinogenicity via bioaccumulation or environmental persistence.

## 📊 Key Improvements Over Baseline

| Dimension | Baseline | Obscura |
|---|---|---|
| Features | 3 descriptors | ~7,000 (ECFP4/6, MACCS, RDKit FP, 200 descriptors) |
| Class imbalance | Unhandled | sensitivity_boost=3.5 + Platt calibration |
| Evaluation | AUC only | AUC, AUPR, MCC, Sensitivity, Specificity, Brier |
| Ensemble | Hardcoded | AUPR-optimized weights on validation set |
| Threshold | Fixed 0.5 | MCC-optimal 0.43 per assay |
| Per-assay models | None | 12 dedicated classifiers with per-assay thresholds |
| Explainability | None | SHAP TreeExplainer, grouped by feature type |

## 🚀 Setup & Installation

### 1. Clone the repository
```bash
git clone https://github.com/aishwariyasaha/Obscura-Drug-Toxicity-Prediction-AI.git
cd Obscura-Drug-Toxicity-Prediction-AI
```

### 2. Install dependencies
```bash
pip install lightgbm xgboost scikit-learn numpy pandas matplotlib seaborn
pip install rdkit shap joblib imbalanced-learn
pip install streamlit pubchempy pillow
```

### 3. Download the dataset
Place the Tox21 dataset at `data/raw/tox21.csv`.
Dataset source: [Kaggle — Tox21 Dataset](https://www.kaggle.com/datasets/epicskills/tox21-dataset)

---

## 🏃 Running the Pipeline

### Step 1 — Train the model
```bash
python run.py
```

Optional flags:
```bash
python run.py --optuna --n-trials 30    # Hyperparameter optimization via Optuna
python run.py --label majority          # Use majority-vote label strategy
python run.py --no-rf                   # Skip Random Forest for faster training
```

This will generate all model artifacts inside `models/` and evaluation plots inside `outputs/`.

### Step 2 — (Optional) Generate visualizations
```bash
python visualize.py --data data/raw/tox21.csv --output outputs/
```

### Step 3 — Launch the prediction interface
```bash
streamlit run app.py
```

### Quick model sanity check
```bash
python test_model.py
```

## 🔬 How It Works — Full Pipeline

```
SMILES Input
     │
     ▼
Molecular Standardization (RDKit)
     │
     ▼
Feature Engineering (~7,564 features)
     ├── ECFP4 / ECFP6 / FCFP4 / MACCS Keys / RDKit FP
     ├── 200 Physicochemical Descriptors (RDKit)
     └── 19 ADMET Rule Flags (Lipinski, Veber, nitro groups, halogens, etc.)
     │
     ▼
Median Imputation + Zero-Variance Feature Removal
     │
     ▼
┌─────────────┬─────────────┬──────────────┐
│  LightGBM   │   XGBoost   │ Random Forest│
│ (calibrated)│             │              │
└──────┬──────┴──────┬──────┴──────┬───────┘
       │             │             │
       └─────────────┴─────────────┘
                     │
              AUPR-Weighted Ensemble
          (weights optimized on validation set)
                     │
                     ▼
            Toxicity Probability
                     │
            MCC-Optimal Threshold (0.430)
                     │
              ┌──────┴──────┐
           TOXIC         NON-TOXIC
```

Additionally, **12 per-assay models** are trained in parallel — one per Tox21 target — each with assay-specific class weights and individually MCC-optimized thresholds.

## 🔬 Feature Engineering in Detail

Each molecule is represented by **7,564 features** across 6 groups:

| Feature Type | Dimensions | Description |
|---|---|---|
| ECFP4 | 2048 | Morgan fingerprint (radius=2, with chirality) |
| ECFP6 | 2048 | Morgan fingerprint (radius=3, with chirality) |
| FCFP4 | 1024 | Feature-based pharmacophore fingerprint |
| MACCS Keys | 167 | Structural keys — toxicophore detection |
| RDKit FP | 2048 | Topological fingerprint |
| RDKit Descriptors | 200 | Full physicochemical descriptor suite |
| ADMET Flags | 19 | Rule-based flags (Lipinski, Veber, nitro, halogens, etc.) |

## 🖥️ Prediction Interface (Streamlit)

The `app.py` interface provides:

- **Name → SMILES lookup** via PubChem (type "aspirin", "caffeine", etc.)
- **Molecule structure visualization**
- **Aggregate toxicity probability** with color-coded risk level
- **ADMET property panel** with Lipinski/Veber rule flags
- **Structural alerts** (nitro groups, halogens, high LogP, etc.)
- **Per-assay bar chart** across all 12 Tox21 targets
- **Highest risk assay** identification

### Risk Levels

| Level | Probability | Indicator |
|---|---|---|
| HIGH | ≥ 0.65 | 🔴 Red |
| MODERATE-HIGH | ≥ 0.45 | 🟠 Orange |
| MODERATE | ≥ 0.30 | 🟡 Yellow |
| LOW-MODERATE | ≥ 0.18 | 🟢 Teal |
| LOW | < 0.18 | 🟢 Teal |

## ⚠️ Limitations

- Predictions are scoped to **Tox21 assay mechanisms only**
- Does not predict carcinogenicity, bioaccumulation, or chronic toxicity
- Model performance is bounded by Tox21 dataset noise and label sparsity
- Not validated for clinical or regulatory use
